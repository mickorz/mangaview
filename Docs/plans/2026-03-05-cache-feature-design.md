# MangaView 缓存功能设计文档

## 概述

为 MangaView 添加书籍缓存和阅读进度记录功能，实现启动时恢复上次阅读状态。

## 需求总结

| 需求项 | 决策 |
|--------|------|
| 缓存内容 | 文件夹文件 + 阅读进度 |
| 存储方式 | IndexedDB 存储 File 对象 |
| 阅读进度 | 页码记录 |
| 启动行为 | 弹窗显示缓存书架，手动选择 |
| 交互方案 | 启动弹窗显示缓存书架 |

## 数据结构设计

### IndexedDB 数据库

**数据库名称:** `MangaViewCache`

**Store 1: books（书籍缓存）**

| 字段 | 类型 | 说明 |
|------|------|------|
| id | string | 书名（主键） |
| files | File[] | 图片文件数组 |
| createdAt | number | 创建时间戳 |
| updatedAt | number | 最后更新时间戳 |

**Store 2: progress（阅读进度）**

| 字段 | 类型 | 说明 |
|------|------|------|
| bookId | string | 书名（主键） |
| pageIndex | number | 当前页码索引 |
| lastReadAt | number | 最后阅读时间戳 |

## 功能流程

### 保存缓存时机

```
用户拖拽文件夹
      |
      v
+------------------+
| 解析文件并分组     |
+------------------+
      |
      v
+------------------+
| 保存到 IndexedDB  |  <-- 自动保存书籍
+------------------+
      |
      v
+------------------+
| 更新阅读进度      |  <-- 滚动/翻页时自动保存当前页码
+------------------+
```

**保存触发点：**
- 拖拽文件夹时 → 保存书籍文件到 IndexedDB
- 切换书籍时 → 保存当前书籍的页码
- 翻页/滚动停止 500ms 后 → 保存当前页码
- 删除书籍时 → 从 IndexedDB 删除对应数据

### 启动恢复流程

```
应用启动
      |
      v
+------------------+
| 检查 IndexedDB   |
+------------------+
      |
      +---> 有缓存数据 ---> 显示缓存书架弹窗
      |                         |
      |                         +---> 用户选择书籍 ---> 加载并跳转到记录页码
      |                         |
      |                         +---> 用户关闭弹窗 ---> 显示空白页面
      |
      +---> 无缓存数据 ---> 显示空白页面（拖拽提示）
```

## UI 界面设计

### 缓存书架弹窗

```
+----------------------------------------------------------+
|  [X]                                          历史记录     |
+----------------------------------------------------------+
|                                                          |
|  +------------+  +------------+  +------------+          |
|  |            |  |            |  |            |          |
|  |  [封面图]   |  |  [封面图]   |  |  [封面图]   |          |
|  |            |  |            |  |            |          |
|  +------------+  +------------+  +------------+          |
|  |    书名1    |  |    书名2    |  |    书名3    |          |
|  +------------+  +------------+  +------------+          |
|                                                          |
|                    [清空所有缓存]                          |
+----------------------------------------------------------+
```

**卡片交互：**
- 点击卡片 → 加载该书籍并跳转到记录的页码
- 悬停显示删除按钮 → 删除该书籍缓存
- "清空所有缓存" → 清空所有数据

**排序：** 最近阅读的排在最前面

  // 初始化数据库

## 实现要点

1. **防抖保存进度** - 滚动时使用 500ms 防抖，避免频繁写入
2. **封面图生成** - 从缓存的 files 中取第一张作为封面
3. **错误处理** - IndexedDB 操作失败时静默处理，不影响正常使用
4. **存储限制** - IndexedDB 有容量限制，大文件可能需要考虑压缩或提示用户
